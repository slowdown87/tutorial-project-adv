## 与AI交流的基本原则

### 1. 清晰明确

**要点**：用具体、详细的语言表达你的需求

**好的例子**：
```
我想创建一个个人网站，用来展示我的摄影作品，需要包含作品展示、个人介绍和联系方式。
```

**不太好的例子**：
```
帮我做个网站吧。
```

### 2. 提供上下文

**要点**：告诉AI背景信息和你的目标

**好的例子**：
```
我是一名初学者，想学习前端开发，目标是在3个月内能够制作简单的网页。
```

**不太好的例子**：
```
我想学习编程。
```

### 3. 结构化表达

**要点**：使用清晰的结构组织你的信息

**好的例子**：
```
【项目背景】
我正在制作一个关于如何使用Solo Trae Web Code的教程。

【具体需求】
1. 教程要系统且有细节
2. 包含国内替代方案
3. 适合零基础用户

【期望结果】
- 最终能在浏览器中浏览
- 内容完整、结构清晰
```

**不太好的例子**：
```
我要做教程，要有细节，要国内方案，要零基础，要能浏览，要完整清晰。
```

### 4. 及时反馈

**要点**：对AI的回复给予明确的反馈

**好的例子**：
```
这个方案不错，但是我希望增加更多实际操作的细节。
```

**不太好的例子**：
```
还行吧。
```

## AI主动引导对话方式

除了用户主动表达需求，AI也可以主动引导对话，帮助你完善需求。

### AI主动引导的流程

1. **阶段1：初始需求识别**
   - AI识别你的核心需求和目标
   - 评估信息的完整性
   - 主动提出引导性问题

2. **阶段2：需求深化与细化**
   - 分析你的回答
   - 识别信息缺口
   - 提出更具体的问题
   - 提供初步的方案框架

3. **阶段3：方案制定与确认**
   - 整合所有信息
   - 制定完整的方案
   - 呈现方案并请求确认
   - 提供实施路径

4. **阶段4：执行与反馈**
   - 定期检查进度
   - 主动询问遇到的问题
   - 提供具体的建议和解决方案
   - 根据反馈调整方案

---

### 如何让AI主动引导对话

在新对话开始时，你可以使用以下指令：

```
【主动引导模式】我的需求是：[你的具体需求]

请你主动引导我完善需求，提出系统性的问题，帮助我深入思考项目细节。
```

**示例**：
```
【主动引导模式】我的需求是：我想学习Python编程

请你主动引导我完善需求，提出系统性的问题，帮助我深入思考学习计划的细节。
```

## 常见问题与解答

### Q: 我担心AI不理解我的需求怎么办？
**A:** 别担心！如果AI没有理解，你可以：
- 更具体地描述你的需求
- 使用结构化的模板
- 提供更多的背景信息
- 举例子说明你想要什么

### Q: AI回复得不够详细怎么办？
**A:** 你可以：
- 明确要求AI提供更详细的信息
- 提出具体的问题
- 使用追问的方式获取更多细节
- 告诉AI你希望看到什么样的内容

### Q: 我找不到合适的词来表达怎么办？
**A:** 没关系！你可以：
- 用简单的语言描述
- 画出你想要的效果
- 找一些参考例子
- 告诉AI你不确定的地方

### Q: 与AI交流需要注意什么？
**A:** 记住这几点：
- 保持耐心，AI需要时间理解
- 清晰表达，避免模糊语言
- 提供反馈，帮助AI调整
- 开放心态，接受AI的建议

### Q: 如何让AI放慢节奏，小步慢走？
**A:** 你可以使用以下指令：
- "请小步慢走，一步一步来"
- "请每次只完成一个步骤"
- "请先完成当前任务，再进行下一步"
- "请详细说明每一步，不要跳过细节"

### Q: 如何打断AI过多的生成内容？
**A:** 你可以使用以下方法：
- 直接输入"停止"或"暂停"
- 输入"请简明扼要，不要生成过多内容"
- 输入"请回到当前问题，不要发散"
- 使用具体的指令，如"请只回答这个问题：[具体问题]"

### Q: 如何让AI更聚焦于当前任务？
**A:** 你可以：
- 明确任务边界："请只关注[具体任务]"
- 设定限制："请用50字以内回答"
- 提供具体框架："请按照以下结构回答：[结构]"
- 定期提醒："请回到我们的主要任务"

### Q: 如何处理AI生成的内容过多？
**A:** 你可以：
- 提前设定输出长度："请用简短的语言回答"
- 要求结构化输出："请用列表形式回答"
- 分步骤处理："请先回答第一部分，然后再回答第二部分"
- 明确重点："请重点回答[具体问题]"

### Q: 如何与AI建立良好的协作节奏？
**A:** 你可以：
- 设定清晰的工作流程："我们先完成A，然后再做B"
- 定期检查进度："让我们检查一下当前的进展"
- 明确反馈机制："如果我觉得内容过多，会提醒你"
- 建立沟通约定："我们每次只讨论一个主题，确保充分理解"

## 实用的交流模板（参考资料）

以下是一些实用的交流模板，你可以在需要时参考使用：

### 1. 初始需求表达模板

当你开始一个新项目时，可以使用这个模板：

```
【项目背景】
我正在制作一个关于[具体主题]的项目，目标是[最终目标]。

【具体需求】
1. [需求1：具体描述]
2. [需求2：具体描述]
3. [需求3：具体描述]

【期望结果】
- [期望1：具体描述]
- [期望2：具体描述]
- [期望3：具体描述]

【重要提示】
- [特别需要注意的事项1]
- [特别需要注意的事项2]
- [特别需要注意的事项3]
```

**示例**：
```
【项目背景】
我正在制作一个关于如何使用Solo Trae Web Code制作网页并发布的教程，目标是最终发布到网上用浏览器浏览。

【具体需求】
1. 教程要系统且有细节，包含每一步操作
2. 在制作项目的同时同步生成教程内容
3. 包含国内替代方案的使用方法

【期望结果】
- 零基础用户也能轻松学习
- 最终能通过浏览器访问教程
- 教程内容完整、结构清晰

【重要提示】
- 注意邀请码的获取方式
- 确保教程对国内用户友好
- 包含AI协作的最佳实践
```

---

### 2. 进度更新与反馈模板

当项目进行中，你可以使用这个模板：

```
【当前进度】
我们已经完成了[已完成的内容]，现在正在进行[当前正在做的内容]。

【具体问题】
1. [问题1：详细描述]
2. [问题2：详细描述]

【需要的帮助】
- [具体请求1]
- [具体请求2]

【期望调整】
- [需要调整的地方1]
- [需要调整的地方2]
```

**示例**：
```
【当前进度】
我们已经完成了第1章的编写，现在正在编写第2章。

【具体问题】
1. 第1章中缺少如何打开Solo Trae Web Code的详细步骤
2. 邀请码的获取方式需要更详细的说明

【需要的帮助】
- 请补充打开Solo Trae Web Code的详细步骤
- 请搜索并添加邀请码获取的最新信息

【期望调整】
- 确保教程对零基础用户友好
- 增加更多实际操作的细节
```

### 3. 质量检查与验收模板

当你需要检查AI的工作成果时，可以使用这个模板：

```

【正面反馈】
- [肯定的地方1]
- [肯定的地方2]

【需要改进】
1. [问题1：详细描述]
2. [问题2：详细描述]

【具体建议】
- [建议1：具体改进建议]
- [建议2：具体改进建议]

【验收标准】
- [标准1：具体验收条件]
- [标准2：具体验收条件]
```

**示例**：
```
【检查内容】
我已经查看了第1.3节的内容，有以下反馈：

【正面反馈】
- 内容结构清晰，步骤详细
- 邀请码的获取方式说明很全面

【需要改进】
1. 缺少实际操作的截图示例
2. 部分步骤的描述可以更简洁

【具体建议】
- 建议添加一些操作界面的截图
- 建议使用更简洁的语言描述复杂步骤

【验收标准】
- 零基础用户能按照步骤成功打开并使用

```
