# AI协作交流模板

## 目标

本模板旨在帮助你与AI助手进行高效、清晰的交流，确保教程制作过程顺利进行。

---

## 交流模板结构

### 1. 初始需求表达

**模板**：
```
【项目背景】
我正在制作一个关于[具体主题]的教程，目标是[最终目标]。

【具体需求】
1. [需求1：具体描述]
2. [需求2：具体描述]
3. [需求3：具体描述]

【期望结果】
- [期望1：具体描述]
- [期望2：具体描述]
- [期望3：具体描述]

【重要提示】
- [特别需要注意的事项1]
- [特别需要注意的事项2]
- [特别需要注意的事项3]
```

**示例**：
```

2. 在制作项目的同时同步生成教程内容

【期望结果】
- 零基础用户也能轻松学习
- 最终能通过浏览器访问教程
- 教程内容完整、结构清晰

【重要提示】
- 注意邀请码的获取方式
- 确保教程对国内用户友好
- 包含AI协作的最佳实践
```

```
- 发生环境
- 重现步骤
- 错误信息（如果有）]

【尝试过的解决方案】
- [解决方案1：描述]
- [解决方案2：描述]

【需要的帮助】
- [具体请求：详细说明需要AI如何帮助]

【期望结果】
- [希望达到的解决效果]
```

**示例**：
```
【问题描述】
在访问Solo Trae Web Code时，遇到了邀请码输入界面，但不知道如何获取邀请码。
- 问题现象：无法进入系统，提示需要邀请码
- 发生环境：首次访问 https://solo.trae.cn/
- 重现步骤：直接访问网站，点击登录
- 错误信息："请输入邀请码"

【尝试过的解决方案】
- 查看网站上的帮助信息，但没有找到邀请码获取方式
- 搜索网上的相关信息，但结果不明确

【需要的帮助】
- 请详细说明邀请码的获取方式和步骤
- 提供官方的申请渠道和注意事项

【期望结果】
- 能够成功获取邀请码并登录系统
- 了解邀请码申请的时间和流程
```

## 交流技巧

### 1. 清晰表达
- **具体而非笼统**：避免使用"帮我做个东西"这样的模糊描述
- **提供上下文**：说明项目背景、目标和约束条件
- **结构化信息**：使用列表、编号等方式组织信息

### 2. 有效反馈
- **及时反馈**：发现问题及时指出，避免问题累积
- **具体反馈**：指出具体问题，而不是泛泛而谈
- **建设性反馈**：不仅指出问题，还提供改进建议

### 3. 高效协作
- **小步快跑**：将大任务分解为小步骤，逐步完成
- **定期检查**：定期检查进度和质量
- **持续改进**：根据反馈不断调整和优化

### 4. 质量保证
- **明确验收标准**：设定明确的质量标准
- **多维度检查**：从不同角度检查内容质量
- **用户视角**：站在目标用户的角度审视内容
