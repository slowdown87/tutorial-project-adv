# AI主动引导对话模板

## 目标

本模板旨在展示如何让AI成为对话的引导者，主动帮助用户完善需求，使方案和规划更加成熟。

---

## 核心思路

### 为什么需要AI主动引导？
- 🎯 帮助用户系统地思考需求
- 📋 确保重要信息不被遗漏
- 🔄 使对话更加高效和结构化
- 📊 让最终方案更加成熟和全面

### AI主动引导的优势
- **减少用户负担**：用户不需要自己组织完整的需求结构
- **提高准确性**：通过系统提问确保信息完整
- **促进思考**：引导用户深入思考项目细节
- **节省时间**：避免反复沟通和澄清

---

## AI主动引导流程

### 阶段1：初始需求识别

**当用户提出初始需求时**，AI应该：
1. 识别核心需求和目标
2. 评估信息的完整性
3. 主动提出引导性问题

**示例**：

**用户**：
"我想做一个教程，教别人如何使用Solo Trae Web Code制作网页"

**AI助手**：
"我很乐意帮你制作这个教程！为了给你提供最全面的方案，我需要了解一些关键信息：

【项目背景】
- 这个教程的目标受众是谁？（初学者、有经验的开发者等）
- 你希望教程最终发布在哪里？（GitHub、Gitee等）

【具体需求】
- 教程需要包含哪些核心内容？
- 你希望教程的详细程度如何？
- 是否需要包含国内替代方案？

【期望结果】
- 你希望通过这个教程达到什么目标？
- 有没有特定的风格或结构要求？

有了这些信息，我就能为你制定一个详细的计划！"

---

### 阶段2：需求深化与细化

**当用户提供初始信息后**，AI应该：
1. 分析用户的回答
2. 识别信息缺口
3. 提出更具体的问题
4. 提供初步的方案框架

**示例**：

**用户**：
"目标受众是初学者，希望发布在网上让大家都能访问，需要包含国内替代方案，详细程度要让零基础用户也能学会。"

**AI助手**：
"非常好！基于你的需求，我已经开始构思教程框架了。为了让方案更完善，我需要了解几个细节：

【内容细节】
- 你希望教程包含哪些具体的网页制作技术？（HTML、CSS、JavaScript等）
- 对于国内替代方案，你更倾向于哪些平台？（Gitee、Cloudflare Pages等）

【结构安排】
- 你希望教程分为几个主要章节？
- 是否需要包含实际的项目案例？

【制作流程】
- 你希望如何同步生成教程？（边做边记录、先完成再整理等）
- 是否需要包含AI协作的最佳实践？

根据你的回答，我会为你制定一个详细的教程大纲和制作计划。"

---

### 阶段3：方案制定与确认

**当用户提供了足够的信息后**，AI应该：
1. 整合所有信息
2. 制定完整的方案
3. 呈现方案并请求确认
4. 提供实施路径

**示例**：

**用户**：
"希望包含HTML、CSS基础，使用Gitee + Cloudflare Pages，分为6-8个章节，包含实际案例，边做边记录，包含AI协作内容。"

**AI助手**：
"收到！基于你提供的所有信息，我为你制定了以下方案：

【教程框架】
1. 第1章：初识Solo Trae Web Code（包括邀请码获取）
2. 第2章：需求分析与规划
3. 第3章：项目初始化
4. 第4章：国内替代方案选择（Gitee + Cloudflare Pages）
5. 第5章：同步教程生成机制
6. 第6章：HTML基础
7. 第7章：CSS样式
8. 第8章：发布与维护

【制作流程】
1. 我们先规划整体结构
2. 然后按照章节顺序制作
3. 每完成一个步骤，同步记录到教程中
4. 最终发布到Gitee + Cloudflare Pages

【特色内容】
- 零基础友好的详细步骤
- 真实的AI协作案例
- 国内替代方案的完整指南
- 实际项目的制作过程

这个方案是否符合你的期望？如果有需要调整的地方，请告诉我！"

---

### 阶段4：执行与反馈

**在项目执行过程中**，AI应该：
1. 定期检查进度
2. 主动询问遇到的问题
3. 提供具体的建议和解决方案
4. 根据反馈调整方案

**示例**：

**AI助手**：
"我们已经完成了第1章的编写，现在正在进行第2章。为了确保教程质量，我想了解：

【当前进度】
- 你对第1章的内容是否满意？
- 有没有需要补充或修改的地方？

【下一步计划】
- 第2章我们将重点介绍需求分析和规划方法
- 你希望这部分包含哪些具体内容？

【潜在挑战】
- 关于邀请码的获取，你是否遇到了什么问题？
- 对国内替代方案的选择，你还有其他考虑吗？

有任何想法或问题，随时告诉我！"

## AI主动引导的最佳实践

### 1. 识别用户意图
- **快速理解**：迅速识别用户的核心需求
- **分类处理**：根据需求类型选择合适的引导策略
- **保持灵活**：根据用户的具体情况调整引导方式

### 2. 结构化提问
- **逻辑顺序**：按照从宏观到微观的顺序提问
- **层次分明**：使用清晰的结构和编号
- **重点突出**：优先询问最重要的信息

### 3. 提供价值
- **初步方案**：基于已有信息提供初步思路
- **专业建议**：分享相关的最佳实践
- **实用资源**：推荐有用的工具和资源

### 4. 持续跟进
- **定期检查**：主动询问项目进展
- **及时反馈**：对用户的问题快速响应
- **调整优化**：根据反馈不断完善方案

## 应用场景示例

### 场景1：用户提出模糊需求

**用户**：
"我想做一个网站"

**AI助手**：
"很高兴听到你想做一个网站！为了给你提供最有价值的帮助，我需要了解一些关键信息：

【项目背景】
- 这个网站的目的是什么？（个人博客、企业网站、电商平台等）
- 你的技术背景如何？（初学者、有经验的开发者等）

【具体需求】
- 你希望网站包含哪些功能？
- 有没有特定的设计风格或参考网站？
- 预算和时间要求是什么？

【技术选择】
- 你倾向于使用什么技术栈？（HTML/CSS/JavaScript、WordPress、React等）
- 是否需要后端功能？

有了这些信息，我就能为你制定一个详细的网站建设方案！"

---

### 场景2：用户需要解决特定问题

**用户**：
"我的网站加载很慢，怎么办？"

**AI助手**：
"网站加载慢确实会影响用户体验！让我帮你分析和解决这个问题。为了提供针对性的建议，我需要了解一些具体情况：

【问题描述】
- 网站的具体加载时间是多少？
- 哪些页面加载特别慢？
- 问题是最近才出现的还是一直存在？

【网站情况】
- 网站使用什么技术栈？
- 有没有使用大型图片或视频？
- 网站托管在哪里？

【已尝试的解决方案】
- 你已经尝试过哪些优化方法？
- 有没有使用缓存或CDN？

有了这些信息，我就能为你提供具体的优化建议！"

---

### 场景3：用户寻求学习指导

**用户**：
"我想学习前端开发"

**AI助手**：
"非常好！前端开发是一个很有前景的领域。为了给你制定最适合的学习计划，我需要了解：

【学习背景】
- 你目前的编程基础如何？（完全新手、有其他语言经验等）
- 学习前端的目的是什么？（兴趣爱好、职业发展等）
- 每天能投入多少时间学习？

【学习目标】
- 你希望达到什么水平？（能做简单网页、找到工作等）
- 对哪些前端技术特别感兴趣？（React、Vue、原生JavaScript等）

【学习偏好】
- 你更喜欢理论学习还是实践项目？
- 有没有特定的学习资源偏好？（视频教程、文档、互动课程等）

基于你的回答，我会为你制定一个个性化的前端学习计划！"

## 总结

AI主动引导对话的核心是：

1. **识别需求**：快速理解用户的核心需求
2. **系统提问**：通过结构化提问完善信息
3. **提供方案**：基于收集的信息制定详细方案
4. **持续跟进**：在执行过程中提供支持和调整

通过这种方式，AI不仅是一个执行工具，更是一个思维伙伴，能够帮助用户：
- 🎯 更清晰地定义目标
- 📋 更系统地规划项目
- 🔄 更高效地解决问题
- 📊 更全面地考虑细节

这种主动引导的对话方式，将大大提升协作效率和最终成果的质量！
