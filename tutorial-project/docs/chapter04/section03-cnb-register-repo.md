# 4.3 GitHub仓库创建与配置

## 目标

在本节中，你将：
- 学习如何在GitHub上创建仓库
- 掌握GitHub仓库的基本设置
- 熟悉GitHub的基本界面
- 了解如何配置GitHub Pages

---

## 步骤1：登录GitHub

### 操作步骤：

1. **访问GitHub官网**
   - 打开浏览器，访问 [https://github.com/](https://github.com/)

2. **登录账号**
   - 点击页面右上角的「Sign in」按钮
   - 输入你的GitHub账号和密码
   - 如果有两步验证，按照提示完成

---

## 步骤2：熟悉GitHub界面

登录成功后，让我们熟悉一下GitHub的主要界面：

### 主要功能区域：
- 🏠 **首页/仪表盘**：查看项目概览和最近活动
- 📁 **Repositories**：管理你的代码仓库
- 👥 **Organizations**：管理组织和团队
- 🔧 **Settings**：个人设置和账号管理

### 导航菜单：
- 左上角是GitHub logo和主菜单
- 右上角是搜索框、通知和用户头像
- 左侧是最近的仓库和活动

---

## 步骤3：创建新仓库

### 操作步骤：

1. **进入仓库创建页面**
   - 登录后，点击页面右上角的「+」号
   - 选择「New repository」

2. **填写仓库信息**
   - **Repository name**：输入 `tutorial-project`
     - 建议使用小写字母和连字符
     - 不要使用空格或特殊字符
   - **Description**：输入「使用Solo Trae Web Code制作的网页教程项目」
     - 简单描述一下这个项目是做什么的
   - **Visibility**：选择「Public」（公开）
     - 公开仓库：任何人都可以查看和克隆
     - 私有仓库：只有你和授权的人可以查看
   - **Initialize this repository with**：可选是否初始化
     - 可以勾选「Add a README file」
     - 也可以留空，稍后自己上传文件

3. **创建仓库**
   - 确认所有信息填写正确
   - 点击「Create repository」按钮
   - 等待仓库创建完成

---

## 步骤4：查看仓库页面

仓库创建成功后，你会看到仓库的主页，这里包含：

### 仓库主页内容：
- 📄 **代码文件**：显示仓库中的文件和目录
- 📋 **README**：项目说明文档（如果初始化时创建了）
- ⭐ **Star**：收藏仓库
- 👁️ **Watch**：关注仓库动态
- 🔀 **Fork**：复制仓库到自己的账号
- 📁 **Clone**：获取仓库的Git地址

### 仓库操作菜单：
- **Code**：查看和管理代码
- **Issues**：跟踪问题和任务
- **Pull requests**：代码审查和合并
- **Actions**：CI/CD流水线
- **Projects**：项目管理
- **Settings**：仓库设置

---

## 步骤5：配置GitHub Pages

现在让我们配置GitHub Pages来托管我们的网站：

1. **进入Settings页面**
   - 在仓库页面，点击顶部的「Settings」标签

2. **找到GitHub Pages设置**
   - 在左侧菜单中，找到并点击「Pages」

3. **配置部署源**
   - 在「Build and deployment」部分，选择「Source」为「Deploy from a branch」
   - 选择「Branch」为「main」
   - 选择「Folder」为「/project」（因为我们的网站文件在project目录中）

4. **保存配置**
   - 点击「Save」按钮
   - 等待GitHub Pages部署完成

5. **获取访问地址**
   - 部署完成后，页面会显示GitHub Pages的访问地址
   - 格式通常为：`https://slowdown87.github.io/tutorial-project/`

## 仓库创建最佳实践

### 1. 命名规范
- 使用有意义的名称
- 避免使用特殊字符
- 建议使用小写字母和连字符
- 保持名称简洁明了

### 2. 描述信息
- 清晰描述项目用途
- 包含关键信息
- 方便他人理解项目

### 3. 初始化选项
- 通常建议初始化README
- 可以添加.gitignore文件
- 根据项目需求选择合适的初始化选项

### 4. GitHub Pages配置
- 选择正确的分支和目录
- 确保有index.html文件
- 等待部署完成后测试访问

## 常见问题与解答

### Q: 公开仓库和私有仓库有什么区别？
**A:** 公开仓库任何人都可以查看和克隆，适合开源项目；私有仓库只有授权用户才能访问，适合私人项目。GitHub Pages对于公开仓库是完全免费的，私有仓库需要GitHub Pro。

### Q: 可以修改仓库名称吗？
**A:** 可以，在仓库设置中可以修改仓库名称，但修改后GitHub Pages的URL也会改变，需要更新相关链接。

### Q: 一个账号可以创建多少个仓库？
**A:** GitHub对于仓库数量没有限制，你可以创建任意数量的公开或私有仓库。

### Q: 可以删除仓库吗？
**A:** 可以，在仓库设置的最底部找到「Delete this repository」选项，但删除操作不可恢复，请谨慎操作。

### Q: GitHub Pages部署需要多长时间？
**A:** 通常需要1-10分钟，部署完成后会在GitHub Pages设置页面显示访问地址。
