2.1 明确我们的目标

目标

在本节中，你将：

  • 明确我们的项目目标
  • 了解需求分析的重要性
  • 学习如何与AI沟通需求
  • 看到真实的AI协作案例

    • ---

    为什么需要明确目标？

    在开始任何项目之前，明确目标是非常重要的。这就像盖房子前要有蓝图一样，没有明确的目标，项目很容易迷失方向。

    明确目标的好处：

  • 🎯 让我们知道要做什么
  • 📋 帮助我们规划步骤
  • 📊 便于衡量项目是否成功
  • 🔄 减少后期的修改和调整

    • ---

    我们的项目目标

    让我们来明确一下我们的项目目标：

    主要目标

  • **制作一个完整的教程**，教别人如何使用Solo Trae Web Code制作网页
  • **最终发布到网上**，让其他人可以通过浏览器访问
  • **同步生成教程**，在制作项目的同时记录每一步

    • 具体需求

  • 教程内容要系统且有细节
  • 包含使用Solo Trae Web Code的步骤
  • 包含如何使用国内替代方案（Gitee + Cloudflare Pages）
  • 最终能在浏览器中浏览

    • ---

    真实案例：如何与AI沟通需求

    下面是我们实际与AI助手沟通的真实对话，这展示了如何清晰地表达需求：

    对话案例1：提出初始需求

    **用户**：

    "我想做一个教程，这个教程就是关于我现在这个项目的，能够在我制作项目的同时同步生成教程，把我的每一步都包含在里面，既系统又有细节。内容是关于如何使用solo trae web code版本来制作一个如何使用GitHub发布网页的教程，最终整个教程能发布在GitHub上用浏览器浏览。你看看怎么实现，给出方案。"

    **AI助手**：

    "我需要先了解当前项目的结构，然后规划一个方案，如何在制作项目的同时同步生成教程，包含每一步的系统和细节。"

    **分析**：

  • 用户清晰地表达了需求：同步生成教程，包含每一步，系统且有细节
  • 明确了内容：使用Solo Trae Web Code，制作如何使用GitHub发布网页的教程
  • 明确了目标：最终发布在GitHub上用浏览器浏览

    • ---

    对话案例2：调整需求

    **用户**：

    "你没明白我的意思，你只给规划方案，真正的教程需要与我制作的项目进度同步"

    **AI助手**：

    "用户需要的是一个规划方案，说明如何在制作项目的同时同步生成教程。我将制定一个详细的方案，包括如何捕获操作步骤、如何组织教程结构、如何确保同步性等。"

    **分析**：

  • 用户指出了AI的理解偏差
  • 强调了"真正的教程需要与项目进度同步"
  • AI及时调整，开始制定详细的同步方案

    • ---

    对话案例3：关于国内替代方案

    **用户**：

    "哦，github好像不太稳定，国内有什么稳定的替代方案"

    **AI助手**：

    "我来为您推荐几个国内稳定的替代方案，同时提供具体的使用指南。"

    **分析**：

  • 用户提出了新的需求：国内稳定的替代方案
  • AI迅速响应，开始研究和推荐替代方案

    • ---

    对话案例4：关注教程的零基础友好性

    **用户**：

    "你着第1章里怎么打开solo web，怎么使用都没教啊，零基础的用户怎么学得会啊"

    **AI助手**：

    "你说得太对了！第1章确实缺少了实际的操作步骤。让我重新修改第1.3节，补充详细的\"如何打开\"和\"如何使用\"的内容，让零基础用户也能看懂。"

    **分析**：

  • 用户指出了教程的不足：缺少零基础用户需要的内容
  • AI承认问题并迅速修改，确保教程对零基础用户友好

    • ---

    对话案例5：关于邀请码的细节

    **用户**：

    "目前账号还需要邀请码你知道这个事吗？你上网搜一下具体信息，教程内容注意细节啊，别太泛泛"

    **AI助手**：

    "好的！让我先上网搜索一下关于 solo.trae.cn 的具体信息，看看邀请码的事情。"

    **分析**：

  • 用户提醒AI注意细节：邀请码的问题
  • 要求AI主动搜索最新信息
  • AI立即执行，搜索相关信息

    • ---

    对话案例6：关于教程的双重价值

    **用户**：

    "你不要我说啥就只做啥，你应该多了解一些相关的最新信息，你说呢？

    另外，我也没做好，说得太粗略，我又想到一点，我这个教程既是怎么做网页的教程，也是怎么使用ai的教程，你说对不对，就像咱俩的交流，是不是也有一些内容是需要编进教程的？就比如咱俩如何一步一步交流，达到双方都舒适的节奏，实现工作协同，你说呢"

    **AI助手**：

    "你说得太对了！这是个非常好的见解！让我先主动搜索更多相关信息，然后我们重新审视教程结构。"

    **分析**：

  • 用户提出了更深层次的需求：教程应该包含"如何使用AI"的内容
  • 强调了真实交流过程的价值
  • AI认同并开始搜索更多信息

    • ---

    从对话中学习：AI协作的最佳实践

    通过以上对话，我们可以总结出与AI协作的几个最佳实践：

    1. 清晰表达需求

  • 具体说明你想要什么
  • 明确目标和要求
  • 提供足够的上下文

    • 2. 及时反馈调整

  • 指出AI的理解偏差
  • 提供具体的改进建议
  • 引导AI朝着正确的方向前进

    • 3. 关注细节

  • 提醒AI注意容易忽略的细节
  • 要求AI提供具体的信息
  • 确保教程内容的准确性

    • 4. 开放思路

  • 提出新的想法和角度
  • 鼓励AI提供更多价值
  • 共同探索更好的解决方案

---

我们的目标确认

经过以上交流，我们的目标更加明确了：

1. **技术目标**：制作一个关于如何使用Solo Trae Web Code制作网页并发布的教程

2. **教学目标**：让零基础用户也能轻松学习

3. **协作目标**：展示如何与AI有效沟通和协作

4. **最终目标**：发布一个完整、系统、有细节的教程网站
