# 项目制作过程捕获方案

## 1. 操作步骤记录

### 实时记录方法
- **文本记录**: 使用Markdown文件实时记录每一步操作
- **语音记录**: 可选使用语音转文字工具，边操作边口述
- **操作日志**: 利用终端命令历史或操作日志工具

### 记录内容
- 操作的具体命令或点击
- 操作的目的和预期效果
- 遇到的问题和解决方法
- 重要的决策点和理由

## 2. 截图捕获

### 截图工具推荐
- **系统自带截图**: Windows (Win+Shift+S), Mac (Cmd+Shift+4)
- **专业工具**: Snagit, Greenshot, Lightshot
- **浏览器插件**: Nimbus Screenshot, Awesome Screenshot

### 截图策略
- **关键步骤必截图**: 每个主要操作步骤都需要截图
- **对比截图**: 操作前后的对比
- **错误场景截图**: 遇到的错误和解决过程
- **细节放大**: 重要的界面元素放大截图

### 截图命名规范
```
images/stepX-actionY-description.png
```
- X: 步骤编号
- Y: 操作编号
- description: 简短描述

## 3. 代码片段保存

### 代码管理
- **版本控制**: 使用git进行代码版本管理
- **代码注释**: 在代码中添加详细注释，说明实现思路
- **代码片段库**: 保存关键代码片段到单独的文件

### 代码展示格式
- 使用Markdown代码块，指定语言
- 关键行添加行号和注释
- 提供完整代码和核心代码两种展示方式

## 4. 过程捕获工具

### 推荐工具
1. **屏幕录制工具**
   - OBS Studio (免费)
   - Camtasia (付费)
   - Loom (在线)

2. **文档同步工具**
   - Notion
   - Obsidian
   - Google Docs

3. **版本控制工具**
   - Git
   - GitHub Desktop

## 5. 捕获流程

### 标准流程
1. **开始操作前**
   - 打开记录文档
   - 准备截图工具
   - 明确该步骤的目标

2. **操作过程中**
   - 执行操作
   - 实时记录操作步骤
   - 捕获关键截图
   - 保存代码片段

3. **操作完成后**
   - 整理记录内容
   - 检查截图质量
   - 验证操作结果
   - 更新教程文档

## 6. 质量控制

### 检查清单
- [ ] 每个步骤都有详细的操作说明
- [ ] 关键操作都有对应的截图
- [ ] 重要代码都有代码片段
- [ ] 操作结果有验证方法
- [ ] 遇到的问题有解决方案

### 审核流程
- 自我审核: 完成一个步骤后立即审核
- 交叉审核: 定期回顾已完成的内容
- 测试验证: 按照教程步骤重新执行，确保可复现性
